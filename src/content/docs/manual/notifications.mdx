# Notifications 教程

ChemVault Notifications 是统一通知与协作网站，集中承载通知、项目消息、文件处理进度、AI 提取结果审核、产品更新和管理员广播。

## 入口

- [https://notify.chemvault.science](https://notify.chemvault.science)
- 登录身份来自 [User Center](https://user.chemvault.science)

## 用户角色

| 角色 | 可做事项 |
| --- | --- |
| 普通用户 | 查看通知、阅读产品更新、管理通知偏好、接收系统通知。 |
| 项目成员 | 查看项目消息、上传项目文件、跟踪 AI 提取任务、审核结果、查看项目活动。 |
| 管理员 | 管理 API Key、Webhook、用户分群、广播、产品更新、反馈和审计日志。 |

## 首页

首页提供快捷入口：

- Notifications
- Messages
- Preferences
- Product updates

页头通知铃会显示通知入口，账号菜单连接 User Center。

## 通知中心

通知中心汇总：

- ChemVault 任务结果。
- 项目消息。
- 系统提醒。
- 管理员公告。
- 服务更新。

常用操作：

1. 按已读状态筛选：all、unread、read。
2. 按通知类型或来源服务筛选。
3. 刷新列表。
4. 单条标记已读。
5. 点击通知跳转到相关页面。
6. 一键全部标记已读。

## 系统通知与 Web Push

用户可以在支持的浏览器启用系统通知。浏览器或环境不支持推送时，站内通知仍可正常使用。

隐私建议：

- 默认通知可以只显示通用提醒。
- 只有允许 preview 的通知才展示更具体正文。
- 不要在系统通知里暴露敏感实验数据、密钥或个人信息。

## 通知偏好

用户可以按类别分别设置站内通知和 Web Push。

常见类别：

- task progress
- task completed
- task failed
- project message
- admin announcement
- system reminder
- security
- billing
- product update
- marketing

## 项目消息

项目消息用于把对话留在项目上下文中。会话列表显示最近消息、更新时间和未读数量。项目成员可以发送消息，AI、任务、系统和管理员消息也可以作为工作流更新出现。

## 项目文件和 AI 任务

Notifications 可以显示项目文件、处理状态和 AI extraction task。

任务阶段包括：

- uploaded
- queued
- processing
- extracting
- validating
- completed
- failed

完成任务可跳转查看提取结果；失败任务会保留面向用户的错误提示。

## 提取结果审核

结果详情页用于人工审核 AI 输出。

可操作：

- 检查结果项。
- 修正字段。
- 接受或拒绝。
- 标记不确定。
- 记录修正原因和审阅意见。
- 批准结果并生成可复用 dataset。
- 导出 JSON、CSV 或 XLSX。

## 产品更新

产品更新展示发布说明、功能改进、维护通知和平台变化。登录用户阅读详情后可标记为已读，也可提交评分和文字反馈。

## 管理功能

管理员可以管理：

- Service API Key
- Webhook events
- Broadcast notifications
- User segments
- Product updates
- Feature update feedback
- Audit logs
