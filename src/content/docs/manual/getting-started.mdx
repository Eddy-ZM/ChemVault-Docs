---
title: 快速开始
description: 按角色选择 ChemVault 的第一次使用路径。
---

如果你第一次使用 ChemVault，先根据自己的角色选择路线。不同产品共享同一套账号和权限，但每个站点承担的任务不同。

## 学生或课程用户

1. 打开 [ChemVault 主站](https://chemvault.science)，从 **Compounds** 进入化合物检索。
2. 用英文名称、分子式、标识符或主题词搜索，例如 `aspirin`、`ketone to alcohol`。
3. 打开结果记录，查看摘要、类型、相关化学背景和链接。
4. 进入 Docs 的化学参考栏目，阅读反应、试剂、实验指南或课程知识点。
5. 需要三维结构时，打开 [Molecule Studio](https://model.chemvault.science/molecule)，用 Search、SMILES 或 PDB 加载结构。

## 研究者

1. 先在 [User Center](https://user.chemvault.science) 登录或注册账号。
2. 在 [Files](https://files.chemvault.science) 上传论文、谱图、CSV、XLSX、报告或手稿。
3. 如果文件需要结构化提取，进入 [Extract](https://app.chemvault.science) 创建项目并上传文档。
4. 运行 AI Extraction 后，不要直接使用结果；先进入 Review Workflow 审核。
5. 审核通过后，将数据导出为 CSV、JSON 或 XLSX。
6. 在 [Notifications](https://notify.chemvault.science) 跟踪任务完成、失败、审核和项目消息。

## 团队管理员

1. 在 User Center 查看用户、角色、权限和服务访问状态。
2. 为成员分配 Extract、Files、Mail、Notifications 或 Molecule Studio 的访问权限。
3. 在 Mail 中管理邮箱地址、角色、邀请码、系统设置和数据看板。
4. 在 Notifications 管理 API Key、Webhook 事件、广播通知、产品更新、反馈和审计日志。
5. 定期检查 Files 的角色权限、分享链接和 Review Flow。

## 开发或集成用户

1. 在 Extract 的 Settings 中创建 API Key，并只授予必要 scope。
2. 使用 Developer API 上传文档、查询项目、创建导出或读取已批准记录。
3. 配置 Webhook 接收 `document.uploaded`、`extraction.completed`、`review.item_approved` 等事件。
4. 在 Notifications 管理服务 API Key 和 Webhook 事件监控。
5. 不要把 API Key、Webhook secret、App Password 或账户密码写入公开文档。

## 第一次使用建议

- 先完成账号登录，再打开 Files、Extract、Notify 等受保护页面。
- 对 AI 提取结果保持人工审核习惯；未审核记录不应作为正式数据使用。
- 用 Files 管理原始文件，用 Extract 管理结构化提取和审核，用 Notifications 跟踪状态。
- 如果你只是在查资料，不需要登录也可以使用主站、中文站、Docs 和 Molecule Studio 的公开功能。

## 20 分钟上手路线

| 时间 | 操作 | 目标 |
| --- | --- | --- |
| 0-3 分钟 | 打开 [产品地图](/manual/product-map/) | 知道每个产品负责什么。 |
| 3-6 分钟 | 登录 User Center | 确认账号、Profile、Security 和服务访问状态。 |
| 6-10 分钟 | 打开 Files 或 Extract | 根据任务上传文件或创建提取项目。 |
| 10-14 分钟 | 打开 Notifications | 确认任务、消息和通知偏好。 |
| 14-17 分钟 | 打开 Mail | 确认邮箱、星标、草稿或 App Password 入口。 |
| 17-20 分钟 | 打开 FAQ 与排障 | 学会处理权限、上传、提取和链接问题。 |

## 使用前准备

使用受保护产品前，建议准备：

- 一个可登录的 ChemVault 账号。
- 需要处理的文件，例如 PDF、DOCX、CSV、XLSX、TXT、谱图或结构文件。
- 清楚的项目名称或文件夹命名方式。
- 如果要审核数据，准备原始文档和判断标准。
- 如果要集成 API，先明确需要的 scope 和回调事件。

## 不同场景的第一个动作

| 场景 | 第一个动作 |
| --- | --- |
| 看公开化学资料 | 打开主站或中文站。 |
| 学习产品怎么用 | 打开 Docs 的使用手册。 |
| 上传研究资料 | 打开 Files。 |
| 从论文提取数据 | 打开 Extract。 |
| 管理账号权限 | 打开 User Center。 |
| 查任务进度 | 打开 Notifications。 |
| 收发邮件 | 打开 Mail。 |
| 画结构或看三维分子 | 打开 Molecule Studio。 |

## 常见新手误区

- 把 Files 当作 Extract：Files 管原始文件，Extract 管结构化提取和审核。
- 把 AI 输出当作正式结论：必须经过 Review Workflow。
- 只开服务访问但忘记页面权限：管理页、审核页可能还需要 Page Access。
- 用主账号密码配置邮件客户端：第三方客户端应使用 App Password。
- 在通知正文中暴露敏感数据：通知应只放必要摘要。
