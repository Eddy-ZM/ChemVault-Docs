| 产品 | 入口 | 主要功能 |
| --- | --- | --- |

## 产品之间的关系

- **User Center 是身份层**：登录、资料、安全、服务权限和管理员操作都从这里统一。
- **Files 是文件层**：谱图、数据集、论文、手稿、实验记录和项目材料放在文件系统里。
- **Extract 是数据提取层**：把论文、报告和表格转成可审核的结构化记录。
- **Notifications 是协作层**：文件处理、提取任务、审核结果、项目消息和产品更新都在这里通知。
- **Mail 是沟通层**：个人邮箱、团队邮箱和第三方邮件客户端配置都在这里。
- **Molecule Studio 是结构查看层**：处理 SMILES、PDB、结构文件和三维可视化。
- **主站和 Docs 是公开说明层**：对外解释功能、教程、知识记录和项目边界。

## 产品分工

| 问题 | 应由哪个产品回答 |
| --- | --- |
| “这个用户能不能进入某个产品？” | User Center |
| “这个文件在哪里、谁上传、能否分享？” | Files |
| “这篇论文里有哪些结构化数据？” | Extract |
| “这个任务完成了吗、失败了吗？” | Notifications |
| “正式通知或附件发给谁？” | Mail |
| “这个分子结构是什么样？” | Molecule Studio |
| “这个概念或化学记录怎么解释？” | 主站、中文站、Docs |

## 入口选择原则

- 只查公开知识：先用主站、中文站和 Docs。
- 需要账号能力：先登录 User Center。
- 文件还没进入系统：先上传到 Files。
- 文件需要变成结构化表格：使用 Extract。
- 需要知道任务状态：查看 Notifications。
- 需要正式沟通：使用 Mail。
- 需要结构图、SMILES、PDB 或三维视图：使用 Molecule Studio。
