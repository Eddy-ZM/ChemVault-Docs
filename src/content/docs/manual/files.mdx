ChemVault Files 是私有研究文件系统，用于管理 spectra、datasets、dossiers、methods、manuscripts 和项目材料。它连接 User Center 账号和权限，并记录文件操作。

## 入口

- 推荐入口：[https://files.chemvault.science](https://files.chemvault.science)
- 登录入口：[https://user.chemvault.science/login](https://user.chemvault.science/login)

如果团队仍在迁移中看到 `file.chemvault.science`，以实际登录跳转和生产配置为准。

## 页面区域

| 区域 | 用途 |
| --- | --- |
| Topbar | 搜索、上传、帮助和账户状态。 |
| Sidebar | Library、Projects & Folders、Tags、Administration 和 Storage。 |
| Workspace | 文件表、Review Flow、Vault Insights。 |
| Inspector | 选中文件的 Details、Preview、Activity 和操作按钮。 |

## 上传文件或文件夹

1. 点击 **Upload files**。
2. 选择 Files 或 Folder。
3. 如果是文件夹上传，原始目录结构会被加入侧边栏。
4. 设置上传访问权限。
5. 拖放文件或选择文件。
6. 等待 upload queue 完成。

适合上传：

- spectra
- datasets
- dossiers
- manuscripts
- reports
- instrument outputs
- spreadsheets

## 推荐命名方式

| 类型 | 命名建议 |
| --- | --- |
| 文献 PDF | `year-author-topic.pdf` |
| 支持信息 | `paper-id-supporting-information.pdf` |
| 谱图 | `project-sample-technique-date.ext` |
| 数据表 | `project-assay-version.xlsx` |
| 手稿 | `project-manuscript-draft-v01.docx` |
| 导出结果 | `project-extract-approved-yyyy-mm-dd.csv` |

命名不应替代权限控制。敏感级别、团队范围和分享范围仍应由角色和访问策略决定。

## 文件浏览

Files 支持两种视图：

- List view：适合看名称、标签、类型、大小、修改时间和批量选择。
- Grid view：适合快速浏览文件夹和文件卡片。

常用筛选：

- All files
- Ready
- Needs review
- Large objects
- Failed

## 标签和文件夹

推荐做法：

- 用 Projects & Folders 表示项目、课程、实验、文献组或客户空间。
- 用 Tags 表示类型和状态，例如 `NMR`、`HPLC`、`draft`、`review`、`released`。
- 不要把敏感权限只靠文件名表达，应使用角色和访问策略。

## Inspector

选择文件后，右侧 Inspector 会显示：

- Details：文件元数据、状态、大小、类型和索引信息。
- Preview：文件预览。
- Activity：文件相关操作记录。
- Download：下载文件。
- Share：创建或查看分享入口。
- Delete：删除或标记删除文件。

## Activity 的用途

Activity 用于追踪文件发生过什么。它适合回答：

- 谁上传了文件。
- 文件是否重新处理过。
- 是否创建过分享链接。
- 是否进入过 Review Flow。
- 是否有失败、删除、恢复或权限相关操作。

## Share

Files 包含只读分享页：

- private share preview
- public share preview

使用建议：

- 外部分享前确认权限范围、有效期和可见内容。
- 分享链接只适合发送给明确对象。
- 重要文件应保留 Activity 和审核记录。

## Review Flow

Review Flow 把文件生命周期分成：

- Intake：文件进入系统。
- Indexed：D1 index 已记录。
- Review：失败上传或需要检查的对象。
- Released：可见性和访问策略确认后进入可用状态。

Review checklist 包括：

- checksum captured
- access policy set
- failed uploads isolated

## Vault Insights

Vault Insights 用于查看：

- storage distribution
- tag concentration
- operational readiness
- largest object
- ready files
- needs review

## 文件状态解释

| 状态 | 含义 | 建议操作 |
| --- | --- | --- |
| Ready | 文件可浏览或可继续处理。 | 可下载、分享或送入下游流程。 |
| Needs review | 文件需要人工检查。 | 查看 Activity 和 Inspector。 |
| Failed | 上传或处理失败。 | 重新上传、换格式或隔离问题文件。 |
| Large object | 文件较大。 | 避免重复上传，确认是否需要分享或提取。 |
| Released | 访问策略确认后的可用文件。 | 可作为项目归档使用。 |

## 权限

Files 使用 User Center 登录身份，并支持角色可见性。常见角色概念包括 owner、internal domain role、research super 和 admin allowlist。

管理员应定期检查：

- 谁能读文件。
- 谁能上传文件。
- 谁能创建分享。
- 谁能删除文件。
- 外部域名是否只能访问明确分享内容。

## 与 Extract 的关系

Files 保存原始文件和项目材料，Extract 负责解析、AI 提取、审核和导出。推荐流程是：

1. 原始资料先进入 Files。
2. 需要结构化的数据再进入 Extract。
3. Extract 导出的 approved dataset 回存 Files。
4. 项目沟通和任务提醒交给 Notifications 或 Mail。
