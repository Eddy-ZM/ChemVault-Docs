The roadmap keeps the documentation system commercially legible: visitors can see what exists now, what is being prepared and which parts are deliberately future-facing.

## Now

| Track | Current focus | Route |
| --- | --- | --- |
| Knowledge library | Chemistry notes, reagent passports and lab guides are grouped into stable entry points. | [Docs Home](/) |
| Commercial positioning | Homepage now explains the ChemVault audience lanes, operating model and next actions. | [Commercial Overview](/projects/commercial-overview/) |
| Interaction layer | Homepage records can be searched and filtered by workspace. | [Active records](/#active-library-records) |
| Course archive | CHEM10712 knowledge points remain indexed under the UoM Chemistry section. | [CHEM10712](/uom-chemistry/chem10712/) |

## Next

- Publish more reagent passports for commonly compared transformations.
- Add printable resource sheets for lab workflows and revision summaries.
- Create curated exam-preparation packs from high-value CHEM10712 notes.
- Strengthen internal links between reaction mechanisms, reagent choice and troubleshooting.

## Later

- Add versioned downloads once the resource metadata workflow is stable.
- Connect public docs to ChemVault Core data models when the platform surface is ready.
- Add release notes for major content imports and public resource drops.
- Expand project manuals into product, content and infrastructure operating playbooks.

## Roadmap rules

- Public promises should stay conservative and useful.
- Planned items should point to a real route or decision owner before publication.
- Completed roadmap work should be reflected in the [Changelog](/projects/changelog/).
- Large content imports should be followed by build, link and search checks.
