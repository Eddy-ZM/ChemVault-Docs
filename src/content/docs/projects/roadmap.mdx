The roadmap keeps the public website focused on product presentation and buyer confidence.

## Now

| Track | Current focus | Route |
| --- | --- | --- |
| SaaS homepage | Product hero, interface preview, feature strip, workflow, security and pricing. | [Home](/) |
| Commercial navigation | Sidebar reduced to buyer-facing pages only. | [Commercial Site](/projects/) |
| Product mock | Display-only workflow, access, asset and routing preview. | [Product preview](/#product) |
| Pricing display | Team, Business and Enterprise tiers for buyer conversations. | [Pricing](/#pricing) |

## Next

- Add a real contact or demo form when the lead-capture path is ready.
- Add customer proof only when real logos or case studies are available.
- Replace display pricing with final packaging once the business model is set.
- Add a dedicated security page if access, audit and data-residency claims need more detail.

## Later

- Connect the marketing site to the production app once the app routes are stable.
- Add product screenshots from the real application instead of static interface previews.
- Add comparison pages only if there is a clear buyer segment and positioning strategy.

## Roadmap rule

Keep the public site commercial and conservative. Do not add internal notes, course material or backend implementation detail to the main buyer journey.
