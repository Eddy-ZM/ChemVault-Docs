---
title: SaaS Operating Model
description: The public SaaS model behind ChemVault's commercial product presentation.
---

ChemVault should be presented as a controlled operating system for chemistry teams. The public website should show a clear product shape without exposing backend implementation or internal project detail.

## Product layers

| Layer | What it shows | Commercial value |
| --- | --- | --- |
| Workspace | Requests, active work, approvals and assets in one surface | Reduces scattered team coordination |
| Workflow control | Status, owners, checkpoints and handoffs | Makes delivery predictable |
| Access control | Roles, permissions and external identity inheritance | Gives teams confidence before onboarding |
| Email routing | Inbound requests routed into the right workspace | Converts communication into trackable work |
| Analytics | Activity, bottlenecks, completions and review load | Gives managers operational visibility |

## Public experience

The site should answer four buyer questions quickly:

1. What does ChemVault do?
2. What does the product interface look like?
3. How does it control access, routing and approvals?
4. How would a team buy or evaluate it?

## What stays out of the main display

- Non-commercial reference material.
- Backend errors, database migrations and private implementation details.
- Over-specific claims that are not yet supported by a real product surface.

The commercial site can still be hosted in the current documentation framework, but the visible route should feel like a normal SaaS product website.
