This matrix keeps the public product story focused on SaaS capabilities rather than background notes.

## Commercial capabilities

| Capability | Display state | Buyer value |
| --- | --- | --- |
| Operations dashboard | Live display | Shows active workflows, approvals, assets and teams at a glance. |
| Workflow tracking | Live display | Turns repeated work into visible status and ownership. |
| Role permissions | Live display | Shows how teams can separate view, edit, approval and admin rights. |
| Email routing | Live display | Explains how inbound requests can become trackable work. |
| Asset control | Live display | Presents controlled files and records as product assets. |
| Analytics | Live display | Shows operational visibility for managers and team leads. |
| Pricing tiers | Live display | Gives buyers a simple Team, Business and Enterprise frame. |
| Contact action | Live display | Sends commercial interest to a clear demo/contact path. |

## Display-only rule

The public website can show product states as a polished interface preview. It should not pretend to execute backend actions from the marketing page. Buttons, toggles and mock controls should be visibly useful as presentation, while real platform actions belong in the app.
