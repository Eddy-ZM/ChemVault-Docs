ChemVault Docs is being shaped as the public operating layer for the ChemVault ecosystem: a place where chemistry knowledge, reagent decisions, lab workflows and product documentation can be searched, maintained and reused.

## Commercial position

ChemVault Docs should communicate three things quickly:

- **Authority** — pages are structured, versioned and connected to real chemistry workflows.
- **Utility** — users can move from a question to a route, table, checklist or decision map.
- **Operations** — the documentation system has owners, release states, roadmap notes and maintenance rules.

## Operating model

| Layer | Purpose | Public route |
| --- | --- | --- |
| Knowledge library | Chemistry notes, reagent passports and lab guides | [Knowledge Library](/chemistry/) |
| Decision workflows | Maps that help users choose routes or reagents | [Reagent Selection Map](/reagents/reagent-selection-map/) |
| Resource library | Templates, download metadata and release planning | [Resource Library](/downloads/resources/) |
| Product operations | Roadmap, changelog and project manuals | [Project Manuals](/projects/) |

## Quality gates

- Every important public resource should have a context page before it is linked.
- Every new page should have a clear audience, route, title and maintenance purpose.
- Decision pages should reduce uncertainty by showing conditions, trade-offs and traps.
- Operational pages should record what changed, why it changed and what remains planned.

## Next commercial moves

- Turn the resource library into a downloadable asset catalogue.
- Add more reagent passports and connect them to reaction decision maps.
- Convert high-value CHEM10712 notes into curated revision packs.
- Keep the changelog and roadmap visible so the site feels maintained rather than static.
