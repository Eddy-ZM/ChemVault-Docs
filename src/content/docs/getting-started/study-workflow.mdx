Use this workflow when you want ChemVault Docs to act like a working desk instead of a folder tree.

The aim is to start with a real task, collect the few pages that matter and finish with an output that can be reused.

## 30-minute study loop

1. Pick one question, not a broad topic.
2. Open the hub page that matches the question.
3. Read one overview page and one detailed page.
4. Write a 5-line answer from memory.
5. Reopen the source page and patch missing assumptions.
6. Add one related link for the next session.

## 60-minute lab loop

1. Define the experiment state: planning, running, monitoring, work-up, purification or reporting.
2. Open the relevant lab guide and reagent passport.
3. Record expected observations before touching the result.
4. Compare the real observation against the expected one.
5. Decide the next action: continue, quench, work up, purify or troubleshoot.
6. Save route, solvent system, Rf values and notes in the file-library format.

## Revision evidence checklist

For every finished revision block, keep:

- the task question
- the three pages used
- the rule or mechanism applied
- one likely exam trap
- one linked follow-up page

## Quick summary

Start from a task, not from the sidebar.

The best ChemVault Docs session ends with a reusable answer, record or checklist.
